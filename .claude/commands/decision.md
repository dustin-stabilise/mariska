Record a decision record (ADR) for: $ARGUMENTS

1. Read `Docs/decisions/0000-template.md` for the canonical format.
2. List `Docs/decisions/` and pick the next sequential number (`DR-0001`, `DR-0002`, …).
3. Fill the template using **only** what this conversation, the repo, or the
   user provides. Leave unknown fields blank rather than inventing content.
   Flag genuine ambiguity with `[UNCLEAR]`.
4. **The Critical Assumption** and **Early Warning Signs** are never skipped —
   if you can't fill them from available facts, ask the user before writing.
5. Set the Review Date ~90 days from today; leave **Outcome** empty.
6. Save as `Docs/decisions/DR-XXXX-<kebab-title>.md` and show the user the
   result. Do not commit unless asked.
