Record a standard operating procedure for: $ARGUMENTS

1. Read `Docs/sops/_template.md` for the canonical format.
2. Fill the template using **only** what this conversation, the repo, or the
   user provides. Leave unknown fields blank rather than inventing content.
   Flag genuine ambiguity with `[UNCLEAR]`.
3. **Verification** and **Common Failures** are mandatory — if you can't fill
   them from available facts, ask the user before writing.
4. Save as `Docs/sops/<kebab-title>.md` and show the user the result. Do not
   commit unless asked.
